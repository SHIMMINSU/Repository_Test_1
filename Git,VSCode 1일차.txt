Git
분산 버전관리 시스템
- 로컬 또는 원격 서버에 같이 프로젝트를 갖고있어 집중형 방식 버전관리의 경우 서버가 죽을시 복구가 어려운 반면
  분산형 방식의 Git의 경우 백업등이 용의하며 여럿이 같은 프로젝트에 작업시 업데이트로 인한 데이터 손실을 방지 가능합니다.
- 코드 비교, 버전 시간별로 관리, 스냅 기억 기능, 중복을 제외한 코드 병합 기능
- 업데이트 절차 작업 > 스테이징 > 커밋(최종)

VSCode
MS에서 제공하는 무료 개발 툴
장점
기존(패키징 프로젝트에 특화) 무거운 VS에 비해 (단일화 프로그램에 특화)VSCode의 경우 가벼우며, 숙달시 응용하여 코드 작성이 매우 빨라집니다.
단점
FiraCode 지원하며, 새로운 글꼴형식을 배워야함, 단축키가 기존 VS와 달라서 적응이 필요.

Git Extensions
Git Bash의 경우 명령어를 사용하는데 명령어를 잘몰라도 사용에 필요한 UI 제공
